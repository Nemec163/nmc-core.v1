# NMC UI Kit

Welcome to the NMC UI Kit documentation.

## Overview

NMC UI Kit is a modern, themeable React component library designed for creating consistent and beautiful interfaces. The library features:

- **Theme Support**: Built-in light and dark themes, plus the ability to create custom themes
- **Component Library**: A collection of carefully crafted React components
- **CSS Variables**: Design tokens implemented as CSS variables for consistent styling
- **Accessibility**: Components designed with accessibility in mind

## Installation

```bash
# Using npm
npm install nmc-ui.v1

# Using yarn
yarn add nmc-ui.v1

# Using pnpm
pnpm add nmc-ui.v1
```

## Basic Usage

1. Import the CSS in your main application file:

```jsx
// For Next.js, in _app.jsx or layout.jsx
import 'nmc-ui.v1/dist/styles/index.css';
```

2. Wrap your application with the ThemeProvider:

```jsx
import { ThemeProvider } from 'nmc-ui.v1';

function App({ Component, pageProps }) {
  return (
    <ThemeProvider defaultTheme="dark">
      <Component {...pageProps} />
    </ThemeProvider>
  );
}
```

3. Use the components in your app:

```jsx
import { Button } from 'nmc-ui.v1';

function MyComponent() {
  return <Button template="red">Click me</Button>;
}
```

## Theming

The UI kit includes a powerful theming system with predefined themes and the ability to create custom themes.

### Default Themes

- `light` - Light theme optimized for day use
- `dark` - Dark theme optimized for night use

### Creating Custom Themes

To create a custom theme:

1. Define your theme variables in CSS/SCSS:

```scss
.my-custom-theme {
  --border-color: #2a2a2a;
  --text-primary: #ffffff;
  --text-secondary: #a0a0a0;
  
  /* Add all required theme variables here */
}
```

2. Register your theme with the ThemeProvider:

```jsx
<ThemeProvider 
  defaultTheme="my-custom-theme" 
  customThemes={['my-custom-theme']}
>
  <App />
</ThemeProvider>
```

## Best Practices

1. **Consistent Theming**: Always use the theme CSS variables instead of hardcoded colors
2. **Component Composition**: Combine simple components to create complex UIs
3. **Accessibility**: Ensure all interactive elements are keyboard accessible
4. **Responsive Design**: Use the responsive variables for consistent layouts across devices
